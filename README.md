## 🔍 О проекте

**Unbound** — это комплексный оркестратор кросс-платформенных технологий обхода систем **Deep Packet Inspection (DPI)**. 
Проект решает фундаментальную проблему современных блокировок, действуя **на уровне пакетов (L3/L4)**, а не абстрактных сетевых туннелей.

> [!IMPORTANT]
> Unbound **не является VPN-сервисом или прокси-туннелем**. Мы не перенаправляем ваш трафик на сторонние серверы. Ваше оригинальное соединение и ваш публичный IP-адрес сохраняются. Unbound просто «переупаковывает» и «фрагментирует» пакеты данных локально, запутывая оборудование провайдера (DPI).

### Ключевые преимущества
1. **Нулевая потеря скорости (Zero Bandwidth Penalty)**: Смотрите 4K видео на YouTube без «просадок» ровно на той скорости, которую предоставляет ваш провайдер.
2. **Нулевой пинг (Zero Latency)**: Отсутствует маршрутизация через океан. Весь трафик идет по кратчайшему пути.
3. **Автономность**: Не нужны подписки, аккаунты, сервера. Движок работает исключительно на мощности вашего устройства.
4. **Легковесность**: Никаких раздутых Electron-приложений. Наш GUI написан на базе **Wails (Go + Vite)**, потребляя минимум RAM.

## 🔬 Архитектура / Как это работает

Системы DPI у провайдеров анализируют проходящие `ClientHello` (напр., SNI в протоколе TLS), чтобы понять, на какой сайт вы идете. Nếu строка совпадает с «черным списком», TCP-пакет сбрасывается (`RST`) или подменяется.

**Unbound использует следующие тактики дезориентации:**
*   **Фрагментация TLS**: Разбивает рукопожатие на микро-части, так что DPI не успевает или не может собрать 100% сигнатуры.
*   **Фейковые пакеты (Spoofing)**: В сеть выбрасывается мусорный пакет с маленьким `TTL`, который оседает прямо внутри оборудования провайдера, сбивая его анализатор состояния соединения.
*   **Смещение окна (Window Size Adjustment)**: Переупорядочивание потоков так, чтобы DPI запутался в `seq`/`ack` номерах.

## 🧰 Совместимость и Движки

Для достижения идеальной работы мы не стали изобретать велосипед для каждой ОС, а бережно интегрировали **самые передовые нативные инструменты перехвата пакетов** на уровне ядра.

| Платформа | Интегрированный Движок | Принцип внедрения | Ограничения |
| :--- | :--- | :--- | :--- |
| **Windows 10/11** | `WinDivert` + `zapret (winws2)` | Прямой перехват пакетов в пространстве ядра. Требует права Администратора. | Необходим запуск `.exe` от имени администратора. |
| **macOS** | `SpoofDPI` | Работает как локальный SOCKS5-прокси, прописываемый через `networksetup`. | Запросы прав через нативный TouchID. |
| **Linux / SteamOS** | `nfqws` + `nftables` | NFQUEUE через подсистему `netfilter`. Тотальный контроль пакетов. | Требует `sudo` при запуске демона. |
| **OpenWrt** | `zapret` (MIPS/ARM) | Сетевые фильтры на уровне входящего роутера. | Прошивка OpenWrt 21.0+ |
| **Android / iOS** | *См. документации платформ* | VpnService (Android), Нативный демон Cydia Substrate/ElleKit (iOS Jailbreak). | iOS требует строгого Rootful/Rootless джейлбрейка. |

## ⬇️ Установка и Скачивание

Все скомпилированные артефакты всегда доступны на нашей [странице релизов GitHub](https://github.com/bobberdolle1/unbound/releases/latest).

### Автоматический запуск (Windows / macOS)
1. Скачайте `Unbound-Windows-v2.0.0.exe` или `Unbound-macOS-Universal.app.zip`.
2. Распакуйте и запустите программу.
3. В интерфейсе выберите **Профиль** (рекомендуется `Unbound Ultimate` для максимальной пробивной способности).
4. Нажмите огромную кнопку **«ПОДКЛЮЧИТЬ»**.

> [!TIP]
> **Автоподбор профиля (Magic Wand 🪄)**: Встроенный алгоритм может в реальном времени пропинговать YouTube, Discord, Twitter и подобрать стратегию обхода с наименьшей задержкой именно под вашего регионального провайдера!

### Linux-инструкция

Вам не нужен графический интерфейс, всё работает через элегантный демон на Rust:
```bash
# 1. Скачайте бинарный файл
wget https://github.com/bobberdolle1/unbound/releases/download/v2.0.0/unbound-linux-amd64-v2.0.0

# 2. Выдайте права
chmod +x unbound-linux-amd64-v2.0.0

# 3. Запустите демон от root'а (в фоне)
sudo ./unbound-linux-amd64-v2.0.0 start
```
*Для завершения сессии пропишите `stop` или `status`.*

## 🏗️ Разработка и Сборка из исходников

Если вы разработчик и хотите модифицировать Unbound (например, добавить свои Lua-скрипты или сменить UI), следуйте этим правилам:

### Требования к среде (Оркестрация конвейера)
- **Golang 1.22+** (для Win/Mac Wails бэкенда)
- **Node.js 18+** (для React-фронтенда и сборки сайта Astro)
- **Rust 1.70+** (для компиляции Linux CLI)
- **Wails CLI** (`go install github.com/wailsapp/wails/v2/cmd/wails@latest`)

### Структура глубоких платформ
Для каждой сложной платформы мы написали **детальную русскоязычную документацию**. Пожалуйста, ознакомьтесь с ними перед тем, как форкать проект:
- 🐧 [Гайд по сборке для Linux (nftables_mgr)](linux/README.md)
- 🤖 [Сборка Kotlin Android APK](android/README.md)
- 🍏 [Оркестрация Theos-твика под iOS (Jailbreak)](theos/unbound-legacy/README.md)
- 🌐 [Специализированная сборка под роутеры OpenWrt](openwrt/README.md)
- 🍎 [Особенности песочницы macOS (`osascript`)](macos/README.md)

### Быстрый билд (Десктоп)

```bash
git clone https://github.com/bobberdolle1/unbound.git
cd unbound
npm install --prefix frontend
wails build -platform windows/amd64,darwin/arm64
```
Или используйте наш кросс-платформенный скрипт-комбайн: `./build_all.sh windows darwin linux`
