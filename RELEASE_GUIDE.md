## 🎯 Как выполнить выпуск

### Требования

Перед запуском скрипта выпуска убедитесь, что у вас есть:

1. **Git** установлен и настроен
2. **GitHub CLI (gh)** установлен и авторизован
   ```bash
   gh auth login
   ```
3. **Go 1.21+** установлен
4. **Wails CLI** установлен
   ```bash
   go install github.com/wailsapp/wails/v2/cmd/wails@latest
   ```
5. **Node.js 18+** установлен (для фронтенда и сайта)
6. Ветка **main** выбрана и обновлена

### Вариант 1: Windows (PowerShell)

```powershell
# Откройте PowerShell от имени администратора
cd F:\Projects\Unbound

# Запустите скрипт выпуска
.\final_launch.ps1
```

### Вариант 2: Linux/macOS (Bash)

```bash
# Перейдите в репозиторий
cd /path/to/Unbound

# Сделать исполняемым (если нужно)
chmod +x final_launch.sh

# Запустите скрипт выпуска
./final_launch.sh
```

## 📋 Что делает скрипт выпуска

### Шаг 1: Предполётные проверки ✅
- Проверяет все необходимые инструменты
- Проверяет авторизацию GitHub CLI
- Подтверждает, что вы в правильной директории
- Проверяет, что вы на ветке main/master

### Шаг 2: Обновление метаданных репозитория ✅
- Устанавливает описание на русском через `gh repo edit`
- Устанавливает URL домашней страницы (GitHub Pages)
- Добавляет 15 тем (dpi-bypass, censorship, zapret, wails...)

### Шаг 3: Сборка всех платформ ✅
- Запускает `scripts/build_all.sh` если доступен
- Резервная ручная сборка:

### Шаг 4: Развёртывание сайта ✅
- Запускает `npm install` в `/website`
- Выполняет `npm run deploy` для обновления GitHub Pages

- Проверяет, существует ли релиз

- Автоматически генерирует заметки о релизе
- Загружает все артефакты из `/dist`
- Оставляет в режиме **ЧЕРНОВИК** для вашей проверки

### Шаг 6: Пост-релизные задачи ✅

- Отправка тега на удалённый репозиторий

## 🔍 Ручной релиз (если скрипт не работает)

Если автоматический скрипт не сработал, вот ручной процесс:

### Сборка всего

```bash
# Десктоп
wails build -platform windows/amd64

# Linux
cd linux && cargo build --release && cd ..

# Android
cd android && ./gradlew assembleRelease && cd ..

# Сайт
cd website && npm run build && npm run deploy && cd ..

# Расширение для браузера
cd extension-web && npm run build && cd ..
```

### Создание релиза вручную

```bash
# Создать директорию dist и собрать артефакты
mkdir -p dist
cp build/bin/unbound.exe dist/
cp linux/target/release/unbound-cli dist/
cp android/app/build/outputs/apk/release/app-release.apk dist/unbound-android.apk
cd extension-web/dist && zip -r ../../dist/unbound-extension.zip . && cd ../..

# Создать релиз
gh release create v2.0.0 \
  --title "Unbound v2.0.0: Глобальное обновление" \
  --draft \
  --generate-notes \
  dist/*

# Тегировать и отправить
git tag -a v2.0.0 -m "Unbound v2.0.0 - Глобальное обновление"
git push origin v2.0.0
```

## 🐛 Диагностика

### «gh auth не найден»
```bash
gh auth login
# Следуйте подсказкам для авторизации
```

### «wails не найден»
```bash
go install github.com/wailsapp/wails/v2/cmd/wails@latest
```

### «Релиз уже существует»
```bash
# Удалить существующий релиз
gh release delete v2.0.0 --yes

# Или отредактировать существующий
gh release edit v2.0.0 --draft=false
```

### «Сборка не удалась на Windows»
- Убедитесь, что запускаете от имени **Администратора**
- Проверьте, что Windows Defender не блокирует winws2.exe
- Убедитесь, что установлены Visual C++ Redistributables

### «Развёртывание сайта не удалось»
- Проверьте, что GitHub Pages включён в настройках репозитория
- Убедитесь, что у вас есть права записи в репозиторий
- Проверьте `website/package.json` на наличие правильного скрипта развёртывания
