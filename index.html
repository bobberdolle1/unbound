Technical Architecture

Packet-level DPI evasion. No tunnels, no proxies. WinDivert intercepts and modifies TCP streams in kernel space.

WinDivert Packet Interception

WinDivert v2.2 kernel driver captures, filters, and modifies raw TCP/UDP packets at the network layer (L3). Winws2 engine performs TCP desynchronization and packet reordering — the core mechanism that breaks DPI signature matching.

Zero Throughput Overhead

Only packets destined for blocked domains are modified. All other traffic — banking, government, CDN — passes through unaltered. No encapsulation, no remote hops, no MTU penalty. Full native bandwidth is preserved.

Parallel Profile Probing

Auto-detection concurrently test-fires profiles against canonical endpoints (YouTube, Discord, Instagram, Twitter, RuTracker). The selection algorithm picks the minimal-modification profile that resolves all targets — no brute force, no guesswork.

GPL-3.0 — Fully Auditable

Every dependency ships inline: Lua DPI-evasion scripts, WinDivert driver binaries, winws2 engine — compiled into a single executable. No hidden fetch calls, no telemetry, no external dependencies. Source is the artifact.

Conflict Detection

Enumerates running services and flags incompatibilities with GoodbyeDPI, Zapret, OpenVPN, Cloudflare WARP, and commercial VPN clients. WinDivert operates at L3 — concurrent packet-level filters cause undefined behavior and are blocked preemptively.

Silent Scheduled Launch

Registers a Windows Task Scheduler entry for UAC-bypassed startup into the system tray. No console window, no elevation prompt on boot. Runs as a background process with the same kernel-level privileges required at install time.

FAQ

Technical Details

What is Unbound — technically?

Unbound is a Go wrapper around the Zapret 2 engine (winws2 binary) and the WinDivert 2.2 kernel packet filter. It provides a system-tray GUI that configures, launches, and monitors winws2 — the same DPI-evasion engine used by GoodbyeDPI and Zapret, but with automated profile selection and conflict detection.

How does the WinDivert engine work?

WinDivert installs a WFP (Windows Filtering Platform) callout driver at L3 (Network Layer) and L4 (Transport Layer). It intercepts raw TCP/UDP packets before they reach the network stack. Winws2 then applies evasion techniques — TCP desynchronization, packet reordering (multidisorder), TTL manipulation, and HTTP/DPI-specific field obfuscation — before reinjecting modified packets. The target server reassembles the stream normally; DPI systems cannot reconstruct the original TLS handshake.

What filter levels does it operate on?

L3 (Network): IP-level filtering by destination/source address. L4 (Transport): TCP/UDP port and flag filtering. WinDivert captures packets at these layers — it does not operate at L7 (Application). DPI evasion is achieved by manipulating transport-layer characteristics (TCP sequence numbers, TTL, window size) that cause DPI parsers to lose stream synchronization while the endpoint TCP stack recovers correctly.

How is this different from a VPN?

A VPN creates an encrypted tunnel to a remote server, routing all traffic through it — adding latency and reducing bandwidth. Unbound modifies only packets destined for domains on the blocklist. Everything else — local banking, government services, CDNs — traverses the network unmodified. Zero overhead for non-blocked traffic. No remote infrastructure involved.

Why are administrator privileges required?

WinDivert is a kernel-mode WFP callout driver. Installing and binding it requires SeLoadDriverPrivilege — an admin-only right. Once installed, the driver persists across reboots. Unbound registers a Task Scheduler entry for silent startup (no UAC prompt on boot) and runs as a background process in the system tray.

What about macOS and Linux support?

macOS: Uses SpoofDPI — a SOCKS5 proxy approach with system network configuration. Linux: Uses nfqws with iptables/nftables NFQUEUE for packet-level manipulation (equivalent to WinDivert on Windows). The Go desktop wrapper handles launch automation on both platforms. A standalone Rust CLI is also available for headless deployment.

What conflicts should I be aware of?

Since WinDivert operates at L3, concurrent packet-level filters cause undefined behavior. Unbound detects and blocks launch when the following are running: GoodbyeDPI, Zapret (nfqws/winws instances), OpenVPN (TAP adapter conflicts), Cloudflare WARP, and commercial VPN clients (ExpressVPN, NordVPN). Close conflicting processes before launching.
